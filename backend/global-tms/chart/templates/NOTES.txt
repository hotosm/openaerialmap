1. chiitiler and NGINX sidecar have been deployed.
2. You can access raster tiles (zoom 0–15) via your Ingress:

   http://{{ .Values.ingress.host }}/<z>/<x>/<y>.png

   Example:
     curl http://{{ .Values.ingress.host }}/10/511/340.png -o tile.png

   Zoom levels >=16 will automatically redirect to the external TiTiler service.

3. Cache status is reported in the X-Cache-Status response header
   (HIT/MISS/EXPIRED/STALE). To sanity-check caching after deploy:
     curl -sI http://{{ .Values.ingress.host }}/5/16/12.png | grep X-Cache

4. Health check endpoint is available at:
   http://{{ .Values.ingress.host }}/health

5. PMTiles freshness
   The PMTiles object is served from a stable S3 key. The 1h chiitiler and
   nginx cache TTLs are expected to handle normal refreshes. If immediate
   freshness is needed, or if S3 ETag invalidation proves unreliable, restart:

     kubectl rollout restart deployment/{{ include "chart.fullname" . }}

6. Follow-up: self-host glyphs.
   .Values.glyphs.url currently uses demotiles.maplibre.org. Move fonts to
   HOTOSM-controlled storage if label rendering needs tighter uptime guarantees.
